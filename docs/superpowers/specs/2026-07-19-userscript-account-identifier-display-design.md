# 油猴脚本账号标识展示设计

## 目标

在油猴脚本左下角悬浮窗中常驻展示当前 `账号标识`，让用户无需展开设置或查看日志即可判断当前页面对应哪个 Boss 用户。

## 界面与数据流

- 在现有状态区的连接状态和执行状态下方新增第三行，显示 `账号：<accountId>`。
- 账号值直接读取现有 `deliveryIdentity.get().accountId`，不新增配置项、缓存或后端字段。
- 文本使用 `textContent` 写入，避免账号标识被解释为页面标记。
- 标识过长时保持单行并显示省略号；该行的原生提示保留完整账号标识，便于悬停查看。
- 设置面板保存账号标识后仍沿用现有页面重载流程，重载后的悬浮窗读取并显示新值。

## 实现边界

- 只修改 `web_script.js` 及其 Node 回归测试。
- 不展示浏览器实例标识 `workerId`，因为本次需求用于识别用户，现有产品语义中该值对应 `accountId`。
- 不调整悬浮窗按钮、设置面板、身份存储方式、心跳载荷或 Dashboard。

## 错误与布局处理

`deliveryIdentity.get()` 已保证首次使用时生成非空账号标识，因此展示层无需新增空值分支。新增文本沿用状态区的最小宽度和溢出策略，不扩大悬浮窗固定宽度，也不允许长标识撑破布局。

## 测试与验收

- 在本地存储预置账号标识后挂载悬浮窗，验证存在独立账号标识元素。
- 验证可见文本为 `账号：<accountId>`，完整值写入该元素的提示文本。
- 验证账号标识行包含单行省略样式，长标识不会改变悬浮窗宽度。
- 运行油猴脚本 Node 测试、语法检查和仓库完整测试，确认现有状态窗行为无回归。
