# 浏览器实例在线状态与油猴日志设计

## 目标

1. Dashboard 的“浏览器实例”区域只展示当前在线实例。
2. 浏览器仍在进行有效控制长轮询时，不因后台标签页心跳定时器节流而被后端误判为离线。
3. 油猴悬浮窗保留“开始/继续、暂停”，增加可展开的本地日志和清空按钮。

## 现状与根因

- 后端持久化 `_workers`，但在线状态来自进程内 `_clients`。`snapshot()` 返回所有登记 worker，Dashboard 当前无条件渲染整个数组，因此历史 worker 会显示为离线卡片。
- 后端在线 TTL 为 30 秒，只由 `/api/runtime/heartbeat` 更新。油猴同时维持 20 秒控制长轮询，但有效长轮询不更新存活时间。浏览器后台节流 5 秒心跳定时器时，控制连接仍有效，后端却可能把实例标为离线。
- 油猴 `Logger` 当前为无 DOM 转发器。日志进入 `ControlAgent.logs` 待上报队列，成功心跳后即从队列移除，因此该队列不能直接作为可见历史。

## 方案

### 后端存活信号

`RuntimeMonitor.desired_control()` 继续执行完整的协议版本、会话 ID 和会话 epoch 校验。只有校验通过且 `_clients` 中存在该 worker 时，才刷新客户端的单调时钟存活时间和公开 `lastSeen`。

长轮询进入等待前和正常返回前都会经过同一校验与刷新路径。旧会话、未知 worker、鉴权失败、网络错误均不能续期。刷新只修改进程内客户端存活态，不写控制状态文件，不改变 desired-state、revision 或 ack。

### Dashboard 实例列表

`normalizedControlState()` 保留完整实例数组，继续供登记统计、账号配额、全局 desired-state 和离线重连同步使用。

`renderControlInstances()` 在渲染边界创建 `onlineInstances`，只为 `online !== false` 的项目生成卡片。摘要改为在线实例数量；没有在线项时显示“暂无在线浏览器实例”。全局控制仍按后端当前语义作用于全部已登记 worker。

### 油猴本地日志

`StatusIndicator` 维护独立的 `logEntries` 环形缓冲，最多 100 条。`ControlAgent.queueLog()` 只规范化一次日志，然后同时写入：

- `ControlAgent.logs`：待上报队列，成功心跳后移除，失败时回队。
- `StatusIndicator.logEntries`：本地可见历史，不受心跳结果影响。

悬浮窗操作区为三列：“开始/继续”“暂停”“日志”。日志按钮通过 `aria-expanded` 控制默认收起的日志区域；区域内提供“清空”，只清除 `logEntries`。每条记录显示时间和消息，警告与错误使用区分色，消息通过 `textContent` 写入以避免 HTML 注入。展开时自动滚动到最新记录。

右键设置入口、后端控制状态机和 Dashboard 集中日志保持不变。不增加结束、重启或退出按钮，不持久化本地日志。

## 错误处理

- 控制轮询的 404、409、鉴权和网络错误沿用现有连接状态处理，不刷新后端存活时间。
- 心跳上传失败时待上报日志继续回队；本地历史不重复追加。
- 清空本地历史不得修改待上报队列。
- 会话在长轮询等待期间被新会话替换时，旧请求重新校验后返回 `stale_session`，且不能刷新存活时间。

## 测试

- Python：有效控制轮询恢复过期实例的在线状态；旧会话不能续期；轮询不触发控制状态持久化。
- Dashboard Node 测试：混合在线/离线数据只渲染在线卡片，空态和摘要文案正确，完整控制数组仍保留。
- 油猴 Node 测试：日志默认收起、展开/收起、清空、100 条上限、文本安全渲染、清空不影响待上报队列。
- 全量：Python 单元/API 测试、JS 语法检查、Node 逻辑/契约测试。
- 浏览器：Dashboard 桌面与窄屏检查；油猴悬浮窗测试页检查布局、展开和清空交互。

## 非目标

- 不删除后端持久化 worker。
- 不修改全局控制的目标集合。
- 不引入 WebSocket 或新的日志接口。
- 不让油猴本地日志跨页面刷新持久化。
