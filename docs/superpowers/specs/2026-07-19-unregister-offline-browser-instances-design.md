# 离线浏览器实例自动注销设计

## 目标

浏览器实例只在心跳有效期内保持登记。实例超过 `client_ttl_seconds` 未上报心跳，或明确上报 `closed` / `exited` 后，从运行时实例表及持久化 `workers` 中注销。之后相同 `workerId` 再次上线时，按新实例登记并恢复默认 `stopped` 期望状态。

## 数据边界

- 注销实例登记及其会话控制状态，不删除账号级策略、账号别名、审计事件或业务日志。
- 状态快照只返回仍在线的已登记实例，因此 `registeredWorkerCount` 与 `connectedClientCount` 一致。
- 清理在生成运行时快照时执行；控制台轮询、运行时接口和心跳响应都会触发快照，因此无需新增后台线程。
- 清理后立即持久化剩余 `workers`，避免服务重启后重新出现离线实例。

## 错误与并发

清理在运行时条件锁内完成，并通过现有原子状态文件替换机制持久化。若持久化失败，沿用当前状态写入异常行为，不提交半完成的内存删除。

## 测试

- 心跳超过 TTL 后，快照不再包含该实例，登记数归零，持久化文件也不再包含该实例。
- 明确上报 `closed` / `exited` 的实例在下一次快照中立即注销。
- 已注销的相同 `workerId` 再次心跳时作为新登记，期望状态为 `stopped`。
- 运行完整后端测试与 Dashboard 契约测试，确认现有控制协议无回归。
