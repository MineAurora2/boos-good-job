# README 设置参数说明设计

## 目标

在项目根目录 `readme.md` 中提供一份完整、可检索的前端面板设置参考，帮助用户理解每个参数的用途、默认值、合法范围、单位、联动关系和生效条件，减少必须阅读源码才能正确配置项目的情况。

## 范围

说明覆盖以下界面和配置：

- 系统管理中的基础资料
- 后端参数
- 浏览器脚本参数
- 防检测随机化参数
- HR 活跃筛选
- 岗位扣星规则
- LLM 接口管理
- Tampermonkey 状态窗连接设置

简历正文编辑和提示词正文编辑属于资源管理，不逐字段解释，但会说明它们与 LLM 功能的关系。

## 文档结构

在 README 的“启动后端”章节中，面板能力简介之后新增“前端面板设置说明”。说明按用户在界面中看到的分组排列，每个参数使用表格记录：

- 面板显示名称
- 对应配置键
- 默认值或合法范围
- 实际运行作用和关键注意事项

参数较多的浏览器脚本设置进一步拆分为基础执行、岗位预加载、防检测随机化和 HR 活跃筛选，避免单张表过长。

## 关键行为说明

- 明确岗位规则从 5 星开始，每颗星折算 20 分，再与 0 至 100 的匹配阈值比较。
- 明确扣星总数使原始星数小于 0 时直接丢弃；恰好为 0 时仍由匹配阈值决定。
- 明确防检测总开关关闭后，乱序、随机跳过、随机等待和随机省略招呼语均不生效。
- 明确 HR 活跃状态是多选白名单，未知文案只有选择“未知”才放行。
- 明确本地 LLM 简历与 BOSS 在线简历序号是两套独立配置。
- 明确 `character`（回复风格）目前只被保存和下发，未接入招呼语生成或 AI 筛选流程，修改它不会影响当前运行结果。
- 明确 `backend.delivery_db_path` 修改后需要重启，其余常规面板参数保存后热加载；浏览器脚本在后续配置同步和执行阶段使用新值。
- 明确 LLM 招呼语与 AI 二次筛选是两个独立开关，并说明接口不可用时的降级行为。

## 内容维护原则

- 只记录源码默认值和后端实际校验范围，不引用开发者本机 `user_config.json` 的个人配置。
- 参数名称同时给出界面中文名和配置键，便于用户在面板、JSON 与日志之间对应。
- 删除 README 现有的“常用自动化配置”重复短段落，相关内容统一归入完整说明。
- 保留现有快速开始和最小使用路径，不改变安装、启动或部署流程。

- 对照 `app/storage/admin_store.py` 检查合法范围与字段约束。
- 对照 `web_script.js`、`app/scoring.py` 和 `app/llm/` 检查参数的实际消费逻辑。
- 检查 Markdown 标题层级、表格列数、重复内容和内部链接。
