# 定时投递运行编排台设计

## 目标

将定时投递区域重做为与现有控制台一致的“运行编排台”：信息密度适中、状态层级清楚、桌面和手机都能一眼看懂当前计划，并且编辑、保存、关闭计划的路径连续可预期。

## 已确认设计方向

- 整个区域只保留“启用计划”一个总开关。
- 计划关闭时只显示标题、运行摘要和总开关，不显示设置区或空白操作栏。
- 计划打开后一次性平铺全部设置，不新增内部开关、手风琴、步骤向导或二次展开。
- 周期使用分段按钮；每周模式直接显示星期按钮；日期范围模式直接显示范围字段。
- 开始时间通过小时/分钟滚轮弹层选择，持续时长只通过 24 小时圆盘拖动或键盘设置。
- 所有更改先进入前端草稿，唯一主操作为“保存并立即应用”。

## 桌面布局

卡片顶部保留标题、动态状态摘要和“启用计划”开关。展开后依次为：

1. 全宽运行状态条，显示运行中、等待下一窗口、暂停或日期范围结束等状态。
2. 双栏工作区。左侧为平铺设置，右侧为紧凑 24 小时圆盘与 `开始 → 结束` 摘要。
3. 左侧设置按视觉分组排列，但不使用嵌套卡片：执行周期、开始时间、模式相关日期、计划预览。
4. 底部操作栏左侧显示草稿/保存状态，右侧只保留“保存并立即应用”。

圆盘桌面宽度控制在约 250–270px，降低当前 300px 圆盘对整张控制台的视觉压制；左侧取消固定大高度，减少每天和工作日模式下的空白。

## 移动布局

680px 以下改为单列：圆盘在上、设置在下、操作栏最后。430px 以下圆盘约 230–240px，周期按钮改为两列。日期和时间弹层保持底部面板行为并允许短屏滚动，不产生横向溢出。

## 状态与交互

- **关闭且已保存**：设置区和操作栏完全隐藏。
- **启用草稿**：设置区展开，底部显示“有未保存的更改”。焦点留在总开关，键盘用户继续 Tab 进入第一项设置。
- **编辑中**：周期、时间、日期和圆盘实时更新计划预览与跨午夜结束时间。
- **保存中**：锁定主按钮并显示“保存中…”，常驻实时区域播报进度。
- **保存成功/失败**：实时区域显示结果；失败保留草稿和所有输入。
- **关闭待保存**：设置区收起，但底部保留“计划将关闭，尚未保存”和唯一保存按钮；保存成功后操作栏消失，焦点回到总开关。

时间滚轮、定时日期范围和报表单日期共用弹层时一次只允许一个 owner。Esc、取消、点击外部和关闭计划都会正确清理 `aria-expanded` 并把焦点返回来源控件。

## 可访问性

- 总开关使用 `aria-controls` 和 `aria-expanded` 控制设置区，收起内容设置 `inert`。
- 持续时长端点维持 `role="slider"`、1–24 范围、方向键与 Home/End。
- 每个可校验控件使用 `aria-describedby` 关联错误文本；失败时设置 `aria-invalid="true"` 并聚焦第一个无效控件。
- 保存反馈实时区域始终留在无障碍树中，空内容只取消外边距，不使用 `display:none`。
- 折叠动画遵循 `prefers-reduced-motion`。

## 视觉系统

沿用现有控制台的深浅主题、cyan 交互色、green 成功色、细边框和 6–8px 圆角。减少大面积发光和装饰性渐变，使用边框、背景明度和字重建立层级。设置区不嵌套卡片，避免与页面其他操作卡片产生不一致的“卡片套卡片”。

## 接口与兼容

后端接口和字段不变，继续提交 `enabled`、`mode`、`startTime`、`durationMinutes`、`weekdays`、`dateStart`、`dateEnd` 和 `applyNow: true`。旧的非整小时分钟值仍只在前端向上取整显示，保存后才写回整小时值。

- 运行 `scripts/run_tests.ps1`，确认 Python、API、用户脚本、Dashboard 语法和前端契约全部通过。
- 使用 Browser 验证深色/浅色、1280px、680px、390px，以及每天、每周、工作日、日期范围、跨午夜、校验失败、关闭待保存和保存反馈。
- 对照已批准的 A 方案可视化检查布局、字体、边框、间距、圆盘占比、弹层定位和移动端溢出。
- 验证完成后关闭本次启动的浏览器标签、可视化伴侣和项目服务进程。
