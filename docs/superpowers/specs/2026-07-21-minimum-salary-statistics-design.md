# 最低月薪统计口径设计

## 背景

统计面板当前将岗位薪资区间的中点写入 `salaryK`，平均月薪、薪资趋势、地区平均薪资和岗位薪资分布均以该字段为统计基准。例如 `5-8K` 当前按 `6.5K` 参与统计。

本次调整将所有月薪统计统一改为最低薪资口径。调整后，`5-8K` 按 `5K` 参与统计，同时继续保留原始薪资区间及上下限。

## 范围

受影响的统计包括：

- 平均月薪指标
- 每日平均薪资趋势
- 省份和城市平均薪资
- 岗位薪资分布及其分档筛选
- 后端统计摘要中的平均薪资

以下行为保持不变：

- 岗位列表展示招聘页面的原始薪资文本
- `salaryMinK` 和 `salaryMaxK` 继续表示区间上下限
- 最低月薪筛选继续检查区间下限
- 最高月薪筛选继续检查区间上限
- 导出继续包含原始薪资、最低薪资和最高薪资
- `salaryMonths` 继续单独记录薪数，不参与月薪统计

## 方案

保留现有 `salaryK` 统计字段和前后端数据契约，将它的赋值由区间中点改为最低月薪。所有统计消费者已经统一读取 `salaryK`，因此无需在多个前端统计函数中分别改用 `salaryMinK`。

该方案可以一次性覆盖所有统计入口，并避免增加新字段或修改接口结构。`salaryK` 在此处表示用于统计的标准月薪值，其值与 `salaryMinK` 相同。

## 数据流

1. 后端解析招聘薪资文本，得到 `salaryMinK`、`salaryMaxK` 和 `salaryMonths`。
2. 将 `salaryK` 设置为 `salaryMinK`。
3. 后端摘要对有效 `salaryK` 求算术平均值。
4. 前端指标、趋势、地图和薪资分布继续消费 `salaryK`。
5. 无法识别的薪资保持为 `None`，不参与统计，并继续计入识别率的分母。

示例：

| 原始薪资 | 最低月薪 | 最高月薪 | 统计基准 | 分布档位 |
| --- | ---: | ---: | ---: | --- |
| `5-8K` | 5K | 8K | 5K | 5-8K |
| `8-12K` | 8K | 12K | 8K | 8-12K |
| `10K` | 10K | 10K | 10K | 8-12K |
| `5-8K·13薪` | 5K | 8K | 5K | 5-8K |

分档继续采用左闭右开边界：`[0,5)`、`[5,8)`、`[8,12)`、`[12,20)`、`[20,+∞)`。

## 页面文案

将“按薪资区间中位数估算”等说明改为“按最低月薪统计”，明确指标口径。地图中的“平均薪资”仍表示当前地区内各岗位最低月薪的算术平均值，不调整组件结构。

## 测试

- 为薪资解析增加回归测试，验证 `5-8K` 的 `salaryK` 为 `5.0`，上下限仍为 `5.0/8.0`。
- 验证单值 K、元/月区间、额外薪数和无法识别薪资的兼容行为。
- 为面板数据聚合增加测试，验证多个岗位的平均月薪按最低薪资求平均。
- 运行完整 Python 与 JavaScript 测试，确认统计口径修改不影响其他功能。

## 非目标

- 不计算年薪或总包。
- 不改变薪资分档区间。
- 不更改薪资筛选字段名称和语义。
- 不迁移或重写历史投递数据；面板加载时会按新口径重新解析历史薪资文本。
