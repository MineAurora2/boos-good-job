# 配额编辑器轮询保持设计

## 目标

修复 Dashboard 中账号配额编辑器在实时控制状态轮询后自动收起的问题。用户点击“修改”后，编辑器与尚未保存的输入值应持续保留，直到用户明确点击“保存”或“取消”。

## 根因

Dashboard 每 3 秒调用一次 `loadControlState()`，随后由 `renderControlCenter()` 重新渲染实例列表。`renderControlInstances()` 使用 `replaceChildren()` 替换全部实例卡，而当前编辑状态只保存在旧 DOM 的 `hidden` 属性和输入框中，所以每次轮询都会恢复默认的收起状态和服务端数值。

## 方案

- 在 Dashboard 页面状态中增加按账号标识保存的配额草稿映射。映射中存在账号即表示该账号处于编辑状态，映射值为当前草稿文本。
- `instanceQuotaMarkup()` 根据草稿映射生成展开或收起的标记，并使用草稿值填充输入框。实时轮询仍照常更新实例状态、心跳、岗位和配额摘要。
- 点击“修改”时创建草稿；输入时持续更新草稿；点击“取消”时删除草稿并按最新服务端数据重绘。
- 保存成功后删除草稿并重绘；保存失败时保留草稿与编辑状态，方便用户直接重试。
- 同一账号可能出现在多个实例卡中。配额是账号级资源，因此这些卡共享同一份草稿和编辑状态。

## 边界

- 不暂停或延长现有 3 秒轮询。
- 不修改账号配额后端接口、合法范围或保存提示。
- 不改变实例卡其他区域的刷新行为。
- 不处理本次修改前已经存在的 Dashboard 契约测试失配。

## 测试与验收

- Node 契约回归测试先证明当前实现无法根据页面草稿恢复展开状态，再验证草稿值、展开状态和事件状态变更。
- 语法检查和相关 Node 测试通过；完整测试结果单独记录既有失败。
- 浏览器中点击“修改”，等待超过一个 3 秒轮询周期，确认输入框仍可见且草稿值未丢失；点击“取消”后确认恢复收起状态。
