# 油猴脚本设置按钮设计

## 目标

将油猴脚本左下角状态窗的设置入口从隐藏的右键操作改为可见按钮。用户无需记忆右键手势，即可在现有操作区打开或收起脚本设置面板。

## 界面与交互

- 在“开始 / 暂停 / 日志”操作区增加“设置”按钮，四个按钮等宽并排。
- 点击“设置”复用现有设置面板；面板关闭时点击会展开，面板展开时再次点击会收起。
- 设置按钮通过 `aria-expanded` 反映面板状态，并通过 `aria-controls` 关联设置面板。
- 现有关闭按钮、取消按钮和 Escape 键仍可关闭设置面板；关闭后同步更新设置按钮的展开状态。
- 删除状态行的右键监听、右键光标和“右键打开脚本设置”提示。状态行只负责显示连接及执行状态。
- 保留 Tampermonkey 菜单中的账号标识和后端连接命令，作为状态窗未挂载时的备用入口。

## 实现边界

- 复用 `StatusIndicator` 的现有设置面板和 `toggleSettings` / `closeSettings` 流程，不改变设置字段、校验、保存和重连逻辑。
- 仅修改 `web_script.js`、对应的 Node 契约测试和 README 使用说明。
- 不修改 Dashboard，也不调整状态窗之外的配置存储方式。

## 错误处理

本次不新增错误分支。保存校验失败、运行中禁止修改连接设置及重连反馈继续使用现有设置面板行为。按钮只负责切换面板可见性。

## 测试与验收

- 状态窗包含“设置”按钮，操作区采用四列等宽布局。
- 初始状态下设置面板隐藏，设置按钮的 `aria-expanded` 为 `false`。
- 点击设置按钮后面板展开，`aria-expanded` 变为 `true`；再次点击后面板收起并恢复为 `false`。
- 状态行不再响应 `contextmenu` 打开设置，也不再显示右键提示或右键光标。
- 通过关闭、取消和 Escape 关闭面板时，设置按钮状态同步恢复。
- README 不再指导用户右键状态窗，改为点击“设置”按钮。
- 运行 `node --test tests/test_web_script.js`，确认油猴脚本现有行为无回归。
